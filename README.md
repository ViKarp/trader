# Trader RL Toolkit

Модульная sandbox-платформа для экспериментов с обучением с подкреплением в многопортфельной торговле. Репозиторий содержит:

- настраиваемую среду `MultiAssetTradingEnv` с обработкой исторических данных и рыночными издержками;
- минимальный, но расширяемый агент `PPOAgent` с разделением дискретных и непрерывных действий;
- утилиты для генерации синтетических наборов данных и запуска демонстрационного тренинга.

## Структура проекта

```
.
├── pyproject.toml          # зависимости и точка входа CLI
├── src/
│   └── trader_rl/
│       ├── __init__.py
│       ├── agents/         # алгоритмы RL (PPO)
│       ├── cli/            # сценарии командной строки
│       ├── data/           # генераторы/загрузчики данных
│       ├── env/            # конфигурации и торговые среды
│       ├── features/       # расчёт технических индикаторов
│       └── models/         # модели и энкодеры признаков
└── first_entry.py          # совместимый с прошлой версией запуск demo
```

Такой layout упрощает добавление новых сред, агентов и экспериментальных скриптов без переписывания существующего кода.

## Быстрый старт

1. **Установка** (рекомендуется `virtualenv`/`conda`):
   ```bash
   pip install -e .[dev]
   ```

2. **Запуск демонстрации** (создаст синтетические данные и обучит PPO-агента на 30 апдейтах):
   ```bash
   trader-rl-demo
   ```
   или напрямую:
   ```bash
   python -m trader_rl.cli.demo
   ```

3. **Повторное использование компонентов**:
   ```python
   from trader_rl import EnvConfig, MultiAssetTradingEnv, PPOAgent, PPOConfig
   from trader_rl.data.synthetic import make_or_load_sample_csv

   df = make_or_load_sample_csv("data.csv", n_symbols=6)
   env = MultiAssetTradingEnv(df, cfg=EnvConfig(feature_set="tech"))
   agent = PPOAgent(env, PPOConfig())
   agent.train(total_updates=10)
   ```

## Возможные направления развития

- подключение реальных источников данных и интерфейсов брокеров;
- расширение набора признаков (order book, альтернативные данные);
- реализация других алгоритмов RL (SAC, DQN, распределённые методы);
- интеграция инструментов мониторинга (Weights & Biases, TensorBoard).
